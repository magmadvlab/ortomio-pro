🚨 CRITICAL FIX: Prevent crashes when tasks array is undefined

## Error Fixed:
Uncaught TypeError: Cannot read properties of undefined (reading 'filter')
at useMemo hooks in planner components

## Root Cause:
The `tasks` array was undefined during initial component mount, but code was calling
`tasks.filter()` without safety checks, causing runtime crashes.

## Changes Made:

### 1. Planner Page (app/app/planner/page.tsx)
- ✅ Added safety check in generateTimelineData(): `if (!tasks || tasks.length === 0) return []`
- ✅ Added safety check in getUpcomingTasks(): `if (!tasks || tasks.length === 0) return []`
- ✅ Protected all badge counts: `tasks ? tasks.filter(t => !t.completed).length : 0`
- ✅ Protected efficiency calculations: `tasks && tasks.length > 0 ? Math.round(...) : 0`
- ✅ Added conditional rendering for timeline: `tasks && tasks.length > 0 ? ... : <EmptyState>`

### 2. Task Calendar (components/planner/TaskCalendar.tsx)
- ✅ Added safety check in getTasksForDate(): `if (!tasks || tasks.length === 0) return []`

### 3. Task List (components/planner/TaskList.tsx)
- ✅ Protected filter operation: `(tasks || []).filter(task => ...)`

### 4. Professional Calendar (components/planner/ProfessionalCalendar.tsx)
- ✅ Added safety check in getTasksForDate(): `if (!tasks || tasks.length === 0) return []`

## Safety Pattern Implemented:

BEFORE (Crashes):
```typescript
const filteredTasks = tasks.filter(task => task.completed) // ❌ TypeError if tasks is undefined
```

AFTER (Safe):
```typescript
// Early return pattern
if (!tasks || tasks.length === 0) return []
const filteredTasks = tasks.filter(task => task.completed)

// Fallback pattern
const filteredTasks = (tasks || []).filter(task => task.completed)

// Conditional rendering
{tasks && tasks.length > 0 ? tasks.filter().map() : <EmptyState>}
```
