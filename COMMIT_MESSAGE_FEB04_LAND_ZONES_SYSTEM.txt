feat: Implement Land Zones System for Crop Rotation and Soil Memory

Implemented a simplified land zones system to manage crop rotation and preserve soil memory independently from field rows. This addresses the real-world scenario where field rows are temporary (destroyed after harvest/tilling) but the land retains memory.

## Database Changes

### New Tables
- `land_zones`: Fixed macro-zones with simple configuration (name + area in hectares)
  - No GPS required, just name and surface area
  - Status: active or resting
  - Optional: soil type, notes
  
- `soil_memory`: Permanent soil memory independent from field rows
  - Linked to land_zone_id (persists even when field rows are deleted)
  - Stores crop history, performance, nitrogen impact
  - Enables AI to learn from zone history

### Schema Updates
- Added `land_zone_id` to `garden_rows` table (optional reference)
- Field rows can now belong to a specific zone

### SQL Functions
- `get_zone_rotation_suggestions(zone_id, years_back)`: Rotation suggestions based on zone history
- `calculate_zone_soil_health(zone_id)`: Calculate soil health score (0-100)
- `get_zone_history(zone_id, years_back)`: Complete zone crop history

## New Services

### landZoneService.ts
Complete TypeScript service for zone management:
- CRUD operations for zones
- Toggle zone status (active ↔ resting)
- Get rotation suggestions
- Calculate soil health
- Get zone history and statistics
- Count and list field rows in zone

## New UI Pages

Zone management page with:

## User Workflow

### Initial Setup (Once)
1. Create fixed zones (e.g., "Zone A" 2ha, "Zone B" 2ha)
2. Configure optional soil type and notes

### Each Season
1. Choose which zone to cultivate (set active)
2. Put other zones to rest
3. Create field rows in active zone
4. Transplant crops normally
5. System records everything in soil_memory linked to zone

### End of Season
1. Harvest crops
2. Delete field rows (if needed for tilling)
3. Switch zone status (Zone A → rest, Zone B → active)
4. **Soil memory persists in Zone A!** ✅

### Consult History
1. View zone history to see all crops planted in that zone
2. Check soil health score and nitrogen balance
3. Get AI rotation suggestions based on zone history

   - Crop diversity scoring

## Technical Details

- RLS policies enabled for all tables
- Cascade delete on zone deletion (removes soil memory)
- SET NULL on field row deletion (preserves soil memory)
- Indexes for performance on common queries
