feat: Complete Crop Rotation and Land Zones Systems

This commit completes two major interconnected systems for professional agricultural management:
1. Field Row Crop Rotation System
2. Land Zones and Soil Memory System

## 🌾 System 1: Field Row Crop Rotation (COMPLETE ✅)

### Database
- New table: `field_row_crop_history`
  - Tracks complete crop history per field row
  - Stores botanical family, performance metrics, environmental context
  - Enables rotation score calculation

- SQL Functions:
  - `calculate_rotation_score()`: Scores rotation quality (1-100)
  - `get_rotation_suggestions()`: AI-powered crop family suggestions
  - `get_field_row_history()`: Complete historical data

### Services
- `fieldRowCropHistoryService.ts`: Complete CRUD and analysis
  - Get history for field row
  - Calculate rotation scores
  - Get rotation suggestions
  - Recognize 8 botanical families automatically

### UI Components
- `FieldRowCropHistoryPanel.tsx`: Two-tab interface
  - Tab 1: Historical timeline with performance metrics
  - Tab 2: Rotation suggestions with scores and reasoning

### Integration
- Automatic registration on transplant from nursery
- Captures planting context (weather, lunar phase, temperature)
- Updates on harvest with yield and quality data

## 🌍 System 2: Land Zones and Soil Memory (BASE COMPLETE ✅)

### Database
- New table: `land_zones`
  - Fixed macro-zones with simple configuration
  - No GPS required (name + area in hectares)
  - Status: active or resting
  - Optional: soil type, notes

- New table: `soil_memory`
  - Permanent soil memory independent from field rows
  - Linked to land_zone_id (persists when rows deleted)
  - Stores complete crop history, nitrogen impact, success scores
  - Enables long-term zone-level learning

- Schema update: `garden_rows.land_zone_id`
  - Field rows can belong to a zone
  - Optional reference (SET NULL on zone delete)

- SQL Functions:
  - `get_zone_rotation_suggestions()`: Zone-level rotation AI
  - `calculate_zone_soil_health()`: Soil health score 0-100
  - `get_zone_history()`: Complete zone crop history

### Services
- `landZoneService.ts`: Complete zone management
  - CRUD operations for zones
  - Toggle zone status (active ↔ resting)
  - Get rotation suggestions
  - Calculate soil health
  - Get zone history and statistics

### UI Pages
- `/app/garden/zones`: Zone management interface
  - Visual cards with color-coded status
  - Soil health display with score
  - Rotation suggestions with nitrogen indicators
  - Zone statistics and field row count
  - Toggle status, view history, delete zone

## 🔄 How Systems Work Together

### Workflow
1. **Create Zones** (once)
   - Define fixed zones (e.g., "Zone A" 2ha, "Zone B" 2ha)
   - Configure soil type and notes

2. **Each Season**
   - Set Zone A to active, Zone B to resting
   - Create field rows in Zone A
   - Transplant crops to field rows
   - System records in both:
     - `field_row_crop_history` (row-level)
     - `soil_memory` (zone-level)

3. **End of Season**
   - Harvest crops
   - Delete field rows (for tilling)
   - Switch zones (A → rest, B → active)
   - **Memory persists in soil_memory!** ✅

4. **Next Season**
   - AI suggests crops for Zone B based on:
     - Zone history (soil_memory)
     - Rotation principles
     - Nitrogen balance
     - Crop diversity

### Key Innovation: Memory Persistence
When field rows are deleted (after tilling):
- `field_row_crop_history` entries remain (for row-specific analysis)
- `soil_memory` entries remain (for zone-level analysis)
- `field_row_id` becomes NULL in soil_memory
- `land_zone_id` stays intact
- **Result**: Long-term soil memory preserved! ✅
